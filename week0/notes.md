# Week 0 - Lecture Notes

## Problem Solving

Programming is the process of taking **input**, processing it with an **algorithm**, and producing an **output**. The goal of programming is not writing code—it's solving problems efficiently.

---

## Binary

Computers understand only **0** and **1** (bits).

- 8 bits = 1 byte
- Each bit represents a power of 2.
- Everything inside a computer is ultimately stored in binary.

Example:

```
Decimal: 13
Binary : 1101
```

---

## Data Representation

Everything is just binary:

- Numbers → Binary
- Text → ASCII / Unicode
- Images → RGB values
- Audio & Video → Binary data

The computer doesn't know what data "means"; it only stores bits.

---

## Algorithms

An algorithm is a sequence of steps used to solve a problem.

A better algorithm reduces execution time.

Example:

- Search page by page → Slow (O(n))
- Divide the search in half each time → Fast (O(log n))

Always think about finding a **better approach**, not just a working one.

---

## Big-O

Big-O measures how an algorithm scales as the input grows.

Common complexities:

- O(1) → Constant
- O(log n) → Very fast
- O(n) → Linear
- O(n²) → Usually slow

Smaller Big-O = Better performance.

---

## Pseudocode

Write the solution in plain English before coding.

Benefits:

- Organizes your thoughts.
- Makes coding easier.
- Helps find logic errors before writing code.

---

## Abstraction

Break a large problem into smaller reusable pieces.

Instead of repeating code:

- Create a function.
- Reuse it whenever needed.

Good programmers reduce repetition.

---

## Programming Building Blocks

Everything in programming is built from a few concepts:

- Functions
- Variables
- Conditionals
- Loops
- Boolean Expressions

Master these and every programming language becomes easier.

---

## Scratch

Scratch teaches programming logic without worrying about syntax.

The concepts learned in Scratch are the same concepts used later in C, Python, JavaScript, and many other languages.

---

## AI

AI should help you understand concepts—not replace learning.

Knowing the fundamentals allows you to use AI effectively instead of depending on it.
